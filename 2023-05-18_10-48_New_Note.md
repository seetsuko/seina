add:新規（ファイル）機能追加

feat:ユーザー向けの機能の追加や変更
fix:ユーザー向けの不具合の修正
refactor:バグの修正や機能の追加ではないコードの変更
style:コードの動作に影響しない、見た目だけの変更（スペース、
                    フォーマット、欠落の修正、セミコロンなど）
test:テストコードの追加や修正
chore:ビルドプロセスやドキュメント生成などの
                    補助ツールやライブラリの変更


動詞,目的語の順で書くと分かりやすい。

ex.
動詞	意味
start	開始
finish	終了
add	追加
Update	更新
Remove	削除
Fix	不具合修正
Improve	改善

What（何をしたか）
Why（なぜその実装をしたか）
